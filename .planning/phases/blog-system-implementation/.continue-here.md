### Decisions made

- **Decided to use [Supabase]** because:
  - Free tier available
  - Built-in authentication system
  - Real-time subscriptions
  - Excellent TypeScript support
  - Edge functions support
  - Perfect for Vercel deployment
  - Alternative: File-based JSON was too simple, PlanetScale requires paid account

- **Decided to use [Tiptap]** because:
  - Modern, ProseMirror-based editor
  - Excellent TypeScript support
  - Extensive extension ecosystem
  - Built-in collaboration features
  - Active development and support
  - Alternative: MDX was simpler but less flexible

- **Decided to use [password authentication]** because:
  - Simple and secure for single admin user
  - No need for complex auth system
  - Easy to set up and maintain
  - Alternative: Supabase Auth would be better but adds complexity

- **Decided to use [Next.js 16 App Router]** because:
  - Server components for better performance
  - App Router is current best practice
  - Better TypeScript support than Pages Router
